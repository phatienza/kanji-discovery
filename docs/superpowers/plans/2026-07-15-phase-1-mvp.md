# Phase 1 MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and verify the offline Java pipeline that emits and validates the N5 world data and KanjiVG assets.

**Architecture:** A single Java 17 `pipeline` Maven module parses source datasets into immutable records, applies external curation, emits JSON, and validates that JSON against hand-curated scenes. JSON is the only delivery contract; no database or server is introduced.

**Tech Stack:** Java 17, Maven, StAX, Jackson Databind, JUnit 5.

## Global Constraints

- Phase 1 only; no React, PWA, backend, workbook, or account code.
- KRADFILE must be decoded explicitly as EUC-JP.
- JMdict and KANJIDIC2 must be streamed.
- Parser input errors fail loudly; malformed records are never silently skipped.
- `parts` is factual normalized KRADFILE decomposition; `recipe` is curated gameplay input or null.
- Emit `data/n5.json`, not a monolithic all-level file.
- Validate dependency order, exact target coverage, duplicate assignment, and word availability.
- Keep non-obvious choices in `DECISIONS.md`.
